Recommender Systems: What are they?

June 12, 2023 · 2 min · Manuel Martinez
Table of Contents
  • What is a recommendation system?
  • Types of Recommendation Systems

This is the first post in what I plan to be a series of very basic introductions to recommendation engines, and the most basic ones at that. I am by no means an expert, or even a middling practitioner, but just a neophyte, so the series will not be exhaustive. The real purpose of these posts is to learn by explaining, but also to allow you to hit the ground running if you want to set up a basic recommendation system for your purposes.

The goal is to go over the very basics of recommendation engines and the different approaches that have been taken in the past. Some approaches are dictated by the nature of the data, while others are constrained by the technological stack available. In each of the approaches we will go over basic implementation details and code up basic examples from scratch using Python.

I originally wanted to do a simple post about alternating-least squares using rank-ordered data, but I decided to give more background information and to do this at the end of the series. In the end we will generate school recommendations for families applying to schools within the context of school choice using a collaborative filtering approach. Let’s get to it.

What is a recommendation system?#

A recommendation system, or recommendation engine, is a subclass of an information filtering system that provides suggestions for items that are most pertinent to a particular entity. 1 Usually, the recommendations are related to e-commerce, music and movie streaming services, and even news aggregators. Basically, you want to present each entity or user with a set of items that are pertinent; and this amounts to finding a mapping between items and users that somehow maximizes the pertinence.

I mentioned pertinence, but didn’t define it. Even worse, pertinence is a very dubious term. Now, you might ask, well what exactly is pertinence? Which is a great question because different approaches to estimating this mapping use different definitions of what pertinence is.

Before going over the approaches, think about how we would do it. What kind of data would we need? Of course the most basic requirement is to have a tuple $(i, j)$ between user $i$ and item $j$.
